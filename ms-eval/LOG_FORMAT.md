# MS Evaluation Log Format

This evaluation suite uses two log streams per run:

1) **MS log** emitted by the master scheduler (`LF_MS_LOG`), using the existing key-value line format.
2) **App log** emitted by the benchmark program (`LF_APP_LOG`), JSONL format.

Both logs are stored under `ms-eval/logs/<exp>/<timestamp>/<run>/`.

## App log (JSONL)
Each line is a JSON object with a `type` field.

### `run_start`
Fields:
- `ts_mono_ns`: monotonic timestamp (ns)
- `experiment`: `e1`, `e2`, or `e3`
- `mode`: `baseline`, `observe`, `intervene`, or `degrade`
- `reactions`: number of reactions (ready set size)
- `steps`: iterations (logical steps)
- `workload_us`: base workload (us)
- `deadline_us`: relative deadline (us)
- `load_factor`: low-criticality load factor (E3)
- `seed`: RNG seed

### `reaction_start`
Fields:
- `ts_mono_ns`: monotonic timestamp (ns)
- `logical_time_ns`: logical time tag (ns)
- `reaction_id`: reaction index generated by the benchmark

### `reaction_end`
Fields:
- `ts_mono_ns`: monotonic timestamp (ns)
- `logical_time_ns`: logical time tag (ns)
- `reaction_id`
- `deadline_us`
- `missed_deadline` (0/1) computed by the benchmark (physical vs logical+deadline)

### `injection`
Fields:
- `ts_mono_ns`
- `logical_time_ns`
- `reaction_id`
- `injected_delay_us`
- `deadline_us`
- `expected_miss` (0/1)

### `run_end`
Fields:
- `ts_mono_ns`

## MS log (key-value lines)
The runtime emits lines like:
```
<mono_ns>,<LEVEL>,event=ready env=0 reaction_index=5 logical=1000000 deadline=5000 is_input=0 updated=0
```

Key events used by the parsers:
- `event=ready`: reaction enters ready set; includes `logical` and `reaction_index`.
- `event=ready_drop`: ready entry removed (e.g., skip degradation).
- `event=pick_next`: scheduler decision; includes `logical` and `candidate`.
- `event=mismatch` / `event=runtime_selected_missing`: guard/fallback violations.
- `event=degrade`: degradation action applied.
